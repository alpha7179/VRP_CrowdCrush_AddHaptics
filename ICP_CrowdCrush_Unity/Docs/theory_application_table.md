| 구분                       | 적용 이론 (Key Theory)                               | 강의 내용 근거 (Class Concept)                                                            | 프로젝트 적용 내용 (Project Implementation)                                                                                                                                                                                                                                                                                                                                                                                        | 기대 효과 (Expected Effect)                                                                                      |
| -------------------------- | ---------------------------------------------------- | ----------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **Cognitive Experience**   | **Place Illusion** (장소 착각)                       | 고품질 그래픽, 물리 법칙, 입체 음향을 통해 사용자가 가상 공간을 실제 장소로 인지하게 함.  | **Environment & Audio** (`Tokyo_Street` Asset)<br>- 실제 도시를 모사한 고품질 3D 에셋(`Tokyo_Street`)과 사실적인 라이팅(Lighting) 적용.<br>- **Spatial Audio**: 거리감과 방향감이 느껴지는 3D 환경음(Ambience)과 군중 소음을 배치하여 청각적 현장감 구현.                                                                                                                                                                          | 가상 공간임을 알고 있음에도 불구하고, 마치 실제 사고 현장(거리)에 와 있다는 강력한 현존감(Presence) 형성.        |
| **Cognitive Experience**   | **Plausibility Illusion** (타당성 착각)              | 가상 환경 내의 사건(Event)들이 논리적이고 사실적으로 반응하여 "상황이 진짜"라고 믿게 함.  | **CrowdAnim**<br>- 주변 군중 NPC들이 단순히 서 있지 않고 상황에 맞는 여러 종류의 애니메이션을 재생.<br>**GameStepManager.cs**<br>- 플레이어의 이동(Move)과 행동(Action)에 따라 실시간으로 변화하는 시나리오 전개.                                                                                                                                                                                                                  | 정적인 더미가 아닌 '살아있는 군중'으로 인식하게 하여 위기 상황의 현실감(Realism) 극대화.                         |
| **Cognitive Experience**   | **Body Ownership / Embodiment** (신체 소유감 / 체화) | 가상의 신체를 자신의 신체로 인식하고(Ownership), 이를 통해 이동 및 행위(Agency)를 수행함. | **Visual: Arm Asset**<br>- 손이나 컨트롤러만 존재하는 모델이 아닌, 팔까지 표현된 에셋을 사용하여 시각적 연속성 확보.<br>**Locomotion: Continuous Move/Turn**<br>- 텔레포트가 아닌 직접 이동방식(`Continuous Move Provider`)을 사용하여 '내가 걷는다'는 감각 형성.<br>**Interaction: Direct & Gesture**<br>- **Direct Interaction**: 기둥을 잡을 때 실제 손을 뻗는 행위.<br>- **Gesture**: 버튼이 아닌 실제 '웅크리기' 자세로 방어. | 사용자의 신체 도식(Body Schema)을 가상 아바타에 일치시켜, 훈련 상황에서의 생존 행동을 내 몸의 경험으로 각인시킴. |
| **UI**<br>(Spatial Design) | **World Space UI** (공간 UI)                         | UI가 화면을 가리는 오버레이(Overlay)가 아닌, 3D 공간 내의 객체로 존재해야 함.             | **UIBillboard.cs**<br>- `UpdateBillboardRotation`: UI 패널이 항상 플레이어를 정면으로 바라보도록 회전하되, 게임 내 표지판처럼 작동.<br>**IngameUIManager.cs** (`IngameCanvas`)<br>- Canvas Render Mode를 `WorldSpace`로 설정하여 HUD가 아닌 전술 장비처럼 연출.                                                                                                                                                                    | 정보 확인 과정에서도 가상 세계와의 시각적 연결이 끊어지지 않음 (Seamless Experience).                            |
| **UI**<br>(Spatial Design) | **Diegetic UI** (디에제틱 UI)                        | 게임 내 환경 요소(예: 손목시계, 벽면 안내판)로 정보를 전달하는 방식.                      | **IngameUIManager.cs** (HUD Layout)<br>- **상단**: 현재 압박 수준을 알려주는 압력 게이지(Pressure Gauge) 배치.<br>- **중앙**: 즉각적인 행동 지령을 위한 미션 지시사항 및 피드백 텍스트 배치.<br>- **하단**: 목표 달성까지 남은 시간을 보여주는 미션 진행바(Progress Bar) 배치.                                                                                                                                                     | 시선의 흐름에 맞춘 배치로 헬멧 HUD와 같은 몰입감을 주며, 직관적으로 현재 상태(위험도/목표)를 인지시킴.           |
| **UX**<br>(Feedback)       | **Multi-modal Feedback** (다중 감각 피드백)          | 시각, 청각, 촉각을 통합적으로 자극하여 정보 전달 효과 극대화.                             | **Integrated Feedback System**<br>- **시각(Visual)**: 압박 수치 증가 시 화면 가장자리가 붉게 물드는 비네팅(Red Vignette) 효과.<br>- **청각(Auditory)**: 위기 상황 시 군중의 비명 소리와 긴박한 환경 사운드(Environmental Sound) 재생.<br>- **촉각(Haptic)**: 기둥 잡기 등 상호작용 수행 시 컨트롤러 진동(Vibration) 피드백.                                                                                                        | 단일 감각에 의존할 때보다 정보 전달 속도가 빠르며, 위기 상황의 긴박함을 본능적으로 체감.                         |
| **UX**<br>(Feedback)       | **Psychophysiological Feedback** (생체 신호 연출)    | 실제 신체 반응(심박수, 눈 깜빡임 등)을 모사하여 심리적 동화 유도.                         | **PressureVignette.cs**<br>- `pulseMagnitude`: 정적인 붉은 화면이 아니라 `Mathf.Sin` 파동을 이용해 심장 박동처럼 화면이 울렁거림.<br>- 실제 사용자의 심장이 빨리 뛰는 듯한 착각(Body Ownership 확장) 유도.                                                                                                                                                                                                                         | 긴장감과 공포감을 기술적으로 증폭시켜 훈련의 진지함을 높임.                                                      |
| **Ray Interaction**        | **Ray Interaction** (원거리 상호작용)                | 손이 닿지 않는 원거리의 물체나 UI를 직관적으로 선택하고 조작.                             | **PlayerManager.cs** (`interactionKeywords` : "Ray Interactor")<br>- **XR Ray Interactor**를 사용하여 중장거리의 UI 패널을 레이저 포인터처럼 쉽게 조작.<br>- 이동이 제한된 밀집 상황에서도 필요한 정보를 빠르게 습득하고 선택 가능.                                                                                                                                                                                                | 정확하고 빠른 조작 편의성을 제공하여 훈련 외적인 조작 스트레스 최소화.                                           |
| **Technical & Safety**     | **Cybersickness Reduction** (멀미 저감)              | 전정 기관과 시각 정보의 불일치를 줄이기 위해 이동 시 시야각(FOV) 축소.                    | **TunnelingVignette** (XR Interaction Toolkit)<br>- `Turn/Move Vignette`: Continuous Locomotion 조작 시 플레이어의 이동 속도 및 회전량에 따라 주변부 시야를 즉각적으로 차단.                                                                                                                                                                                                                                                       | 자유 이동(Continuous Move)의 높은 자유도를 유지하면서도 멀미 부작용을 기술적으로 억제함.                         |
